Swiper 슬라이더 강의자료
(스와이퍼 메서드별 단계 & 디버깅 설명)

이 문서는 swiper-horizontal2-simple.html에서 Swiper의 주요 메서드가 실제로 어떻게 사용되는지, 함수 실행 흐름과 디버깅 코드를 단계별로 초보자도 쉽게 이해할 수 있도록 설명합니다.

1단계: 슬라이더 이동 메서드 slideTo()와 slideNext()

slideTo(인덱스, 속도) : 원하는 슬라이드 번호(인덱스)로 이동합니다.
slideNext() : 다음 슬라이드로 이동합니다.

예시 코드: 
vSwiper.slideTo(3, 0); // 4번째(인덱스 3) 슬라이드로 즉시 이동
hSwiper.slideTo(0, 0); // 가로 슬라이더 첫 슬라이드로 이동
vSwiper.slideNext();   // 다음(아래) 슬라이드로 이동
실행 흐름 설명:
  • slideTo(3, 0)은 4번째 슬라이드로 바로 이동합니다. (0은 애니메이션 없이 즉시 이동)
  • slideNext()는 현재 위치에서 다음 슬라이드로 이동합니다.
디버깅 코드 예시: 
console.log('[onDone] vSwiper.slideTo(3) 호출');
이렇게 로그를 남기면, 브라우저 콘솔에서 함수가 실제로 실행되는 시점을 확인할 수 있습니다.

2단계: 슬라이더 상태 제어 allowTouchMove와 mousewheel.enable()/disable()

allowTouchMove : 슬라이더를 손가락/마우스로 움직일 수 있는지 여부를 설정합니다.
mousewheel.enable() : 마우스 휠로 슬라이더를 움직일 수 있게 합니다.
mousewheel.disable() : 마우스 휠로 슬라이더를 움직이지 못하게 막습니다.

예시 코드: 
vSwiper.allowTouchMove = false; // 세로 슬라이더 잠금
vSwiper.mousewheel.disable();   // 마우스 휠로 세로 슬라이더 이동 불가
vSwiper.allowTouchMove = true;  // 세로 슬라이더 다시 활성화
vSwiper.mousewheel.enable();    // 마우스 휠로 이동 가능
실행 흐름 설명:
  • 가로 슬라이더 섹션에 들어오면 세로 슬라이더를 잠가서, 사용자가 실수로 위아래로 움직이지 못하게 합니다.
  • 가로 슬라이더를 벗어나면 다시 세로 슬라이더를 활성화합니다.
디버깅 코드 예시: 
console.log('[onChg] vSwiper.allowTouchMove = false, mousewheel.disable()');
이렇게 로그를 남기면, 슬라이더가 언제 잠기고 풀리는지 쉽게 알 수 있습니다.

3단계: 슬라이더 상태 갱신 update()

update() : 슬라이더의 내부 상태(슬라이드 개수, 위치 등)를 새로고침합니다.

예시 코드: 
hSwiper.update(); // 가로 슬라이더 상태 갱신
실행 흐름 설명:
  • 세로 슬라이더가 가로 슬라이더 섹션에 도달하면, 가로 슬라이더의 상태를 새로고침하여 올바르게 동작하도록 합니다.
디버깅 코드 예시: 
console.log('[onVEnd] hSwiper.update() 호출');
상태 갱신 시점에 로그를 남기면, 슬라이더가 언제 새로고침되는지 알 수 있습니다.

4단계: 함수 실행 흐름 예시 (전체 시나리오)

  1. 세로 슬라이더(vSwiper)로 4번째(가로 슬라이더) 섹션에 진입
    console.log('[onChg] 가로 슬라이더 섹션 진입, 세로 슬라이더 잠금');
    • vSwiper.allowTouchMove = false, vSwiper.mousewheel.disable() 실행
    • hSwiper.slideTo(0, 0)으로 가로 슬라이더 첫 슬라이드로 이동
  2. 가로 슬라이더(hSwiper)에서 마지막 슬라이드에서 오른쪽으로 넘김
    console.log('[onDone] 마지막 슬라이드, 세로 슬라이더 활성화');
    • vSwiper.allowTouchMove = true, vSwiper.mousewheel.enable() 실행
    • vSwiper.slideNext()로 5번째(아래) 섹션으로 이동
  3. 가로 슬라이더(hSwiper)에서 첫 슬라이드에서 왼쪽으로 넘김
    console.log('[onDone] 첫 슬라이드, 세로 슬라이더 활성화');
    • vSwiper.allowTouchMove = true, vSwiper.mousewheel.enable() 실행
    • vSwiper.slideTo(2)로 3번째(위) 섹션으로 이동
  4. 5번째에서 4번째로 올라올 때
    console.log('[onVStart] 5→4, 가로 슬라이더 마지막 슬라이드로 이동');
    • hSwiper.slideTo(3, 0) 실행

5단계: 디버깅 팁 & 초보자용 안내

  • console.log()는 함수가 실행될 때마다 메시지를 브라우저의 개발자도구(콘솔)에 출력합니다.
  • 각 단계별로 로그를 남기면, 어떤 함수가 언제 실행되는지를 눈으로 확인할 수 있습니다.
  • 슬라이더가 이동하거나, 잠기거나, 상태가 바뀔 때마다 로그를 확인해보세요.
  • 이렇게 하면 자바스크립트를 잘 모르는 사람도 함수 실행 흐름을 쉽게 따라갈 수 있습니다.
console.log('[onChg] vSwiper.allowTouchMove = false, mousewheel.disable()');
console.log('[onDone] vSwiper.slideNext() 호출');
console.log('[onVEnd] hSwiper.update() 호출');

이 자료를 참고하여, 실제 코드를 실행하고 콘솔 로그를 확인해보면 함수 실행 흐름과 슬라이더 동작 원리를 쉽게 익힐 수 있습니다!
